# Implementation Plan: Portfolio Showcase Site

**Branch**: `001-portfolio-showcase` | **Date**: 2026-04-17 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-portfolio-showcase/spec.md`

## Summary

A statically generated portfolio site that showcases projects in three explicit categories — Personal, Startup, Corporate — with project content authored as Markdown files using a defined template. The site is built with Astro (zero-JS by default, first-class content collections, ideal for static portfolios), styled with Tailwind CSS, deployed to GitHub Pages via GitHub Actions, and meets WCAG 2.1 AA accessibility plus a Lighthouse performance score of 90+.

## Technical Context

**Language/Version**: TypeScript 5.x (Node.js 20 LTS)  
**Primary Dependencies**: Astro 5.x, Tailwind CSS 4.x, `@astrojs/tailwind`, `@astrojs/sitemap`, Astro Content Collections (Markdown + Zod schemas)  
**Storage**: Filesystem — Markdown files under `src/content/projects/{personal,startup,corporate}/`; image assets under `public/projects/<slug>/`  
**Testing**: Vitest for unit/schema validation; Playwright for end-to-end smoke tests; `axe-core` (via Playwright) for accessibility; Lighthouse CI for performance budgets  
**Target Platform**: Modern evergreen browsers (Chrome, Firefox, Safari, Edge — last 2 versions); deployed as static HTML/CSS/JS on GitHub Pages  
**Project Type**: Static web application (single project, no backend)  
**Performance Goals**: Lighthouse Performance ≥ 90 (mobile), LCP < 2.5s on 3G, total page weight < 500KB on landing page  
**Constraints**: No server-side runtime, no tracking pixels; respect `prefers-reduced-motion`; WCAG 2.1 AA compliance; build must fail on schema validation errors or slug collisions  
**Scale/Scope**: ~10–50 projects across 3 categories at v1; ~5 page templates (home, category, project detail, 404, about)
