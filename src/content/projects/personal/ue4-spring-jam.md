## Overview

A weekend Unreal Engine 4 jam in May 2020 with Sebastian Avena. We
paired in person on level design and Blueprint scripting on Sebastian's
machine — the GitHub repo was created mostly to track the project
shell and our `.gitignore` for the UE4 folder layout, while the actual
content lived in the editor's binary assets that never made it into a
shareable build.

The jam ended the same day it started: scope ballooned, a core
Blueprint refused to behave, and we cut our losses rather than
shipping a broken submission. The repo survives as a marker — a
reminder that "we tried this" is a perfectly valid outcome of a
48-hour creative constraint.

Source code lives in a private repository.

## Highlights

- **Pair-programmed in person** — sat next to Sebastian for the full
  session, switching between level design (his strength) and Blueprint
  scripting (mine), with the editor as the shared workspace.
- **Honest scope kill** — when the core mechanic refused to converge,
  we chose not to ship a half-broken jam build rather than burn another
  night on it.
- **A first contact with UE4** — the lasting value wasn't the build,
  it was getting hands-on with Blueprints and the UE4 asset pipeline
  for the first time, which fed directly into later 3D work.

## Lessons Learned

Game jams are a brutally efficient way to discover where a tool's
learning curve actually bites. UE4 Blueprints are forgiving until they
aren't — and the moment you fight the engine instead of the design,
the jam clock is already lost. The takeaway wasn't "we shipped a
game"; it was "we now know which 30 % of UE4 we'd skip on a future
attempt", and that the cheapest thing a small team can ship is an
honest scope-kill instead of a broken artefact.
